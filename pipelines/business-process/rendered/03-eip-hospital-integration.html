Enterprise Integration Patterns Applied
Message Channel
Point-to-point (MLLP/TCP) for HL7v2; pub/sub (Kafka) for FHIR event streams
Content-Based Router
Inspects MSH-9 segment field to determine routing. Mirth channel filter + destination router.
Message Translator
HL7v2-to-FHIR R4 mapping via HAPI FHIR library. PID→Patient, PV1→Encounter, OBX→Observation.
Dead Letter Channel
Failed messages after 3 retries routed to DLQ. Alerts via PagerDuty for manual investigation.
Wire Tap
All messages copied to immutable audit log for HIPAA compliance. 7-year retention.
Competing Consumers
2 instances consuming from Bed Management queue for load balancing during peak admits (40-60/day).
Integration Engine — Mirth Connect 4.6
Inbound Protocols
• MLLP/TCP (HL7v2) — port 2575
• REST/HTTPS (FHIR R4) — port 8443
• Kafka Consumer (event streams)
Outbound Protocols
• REST → Insurance APIs (270/271)
• MLLP → Legacy systems (Cerner, lab)
• Azure Service Bus → Cloud consumers
Throughput

Reliability
