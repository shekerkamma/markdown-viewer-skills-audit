Diagram 5a
Class Diagram — EHR Domain Model
Patient
- id: UUID
- name: HumanName
- dob: Date
- mrn: String
- insuranceId: String
+ getActiveEncounters(): List<Encounter>
+ verifyInsurance(): CoverageStatus
InsurancePolicy
- policyId: String
- provider: Organization
- planType: PlanType
- copay: Money
- status: CoverageStatus
+ checkEligibility(): EligibilityResponse
+ submitClaim(encounter): ClaimResponse
IntakeRecord
- recordId: UUID
- patientId: UUID
- triageLevel: ESI (1-5)
- chiefComplaint: CodeableConcept
- vitalSigns: Observation[]
- arrivalTime: Instant
+ escalate(level): void
+ assignBed(dept): BedAssignment
BedAssignment
- assignmentId: UUID
- bedId: String
- patientId: UUID
- department: Department
- assignedAt: Instant
- releasedAt: Instant?
+ release(): void
+ transfer(newBed): BedAssignment
Patient 1 ────── * IntakeRecord patient has many intake visits
Patient 1 ────── 1 InsurancePolicy active coverage (FHIR Coverage resource)
IntakeRecord 1 ────── 0..1 BedAssignment assigned after triage (if admitted)
